## 四、FPGA 通信引脚推荐

如需向 FPGA 模块发送信息，推荐以下方案：

### 方案 A：UART 方案（推荐，简单可靠）

| GPIO | 用途 | 说明 |
|---|---|---|
| **GPIO 0** | UART1 TX（FPGA 接收） | 空闲，可配置 Serial1 |
| **GPIO 4** | UART1 RX（FPGA 发送） | 空闲，可配置 Serial1 |

> 初始化方式：
> ```cpp
> Serial1.begin(115200, SERIAL_8N1, 4, 0); // RX=GPIO4, TX=GPIO0
> ```
> 发送数据：`Serial1.write(data);` 或 `Serial1.println(message);`

### 方案 B：SPI 方案（高速，适合大数据量）

| GPIO | 用途 | 说明 |
|---|---|---|
| **GPIO 18** | VSPI MOSI（主出从入） | 可复用为 FPGA MOSI |
| **GPIO 23** | VSPI MOSI 备选 / 通用 | 可配置为 CS 或时钟 |
| **GPIO 19** | VSPI MISO（主入从出） | FPGA 回复数据 |
| **GPIO 5** | 片选（CS） | 可作为 FPGA 片选 |
| **GPIO 19** | 可复用为 HSPI | 触摸屏 HSPI 已使用，但可重新分配 |

> HSPI 总线已被触摸屏占用，若需高速 SPI 通信，可将触摸屏移至其他 GPIO。

### 方案 C：GPIO 简单脉冲方案（引脚最少）

| GPIO | 用途 | 说明 |
|---|---|---|
| **GPIO 4** | 数据线（单比特发送） | 配合时钟引脚或固定节拍 |
| **GPIO 5** | 可选：使能/握手信号 | 通知 FPGA 数据有效 |

## 五、特殊说明

- **GPIO 6~11**：SPI Flash 专用，**严禁使用**
- **GPIO 34, 35, 36, 37, 38, 39**：仅输入引脚（input-only），不能输出
- **GPIO 34** 目前空闲，可作为 FPGA 读取 ESP32 状态的输入
- **GPIO 0**：上电时必须为高电平以正常启动，进低电平则进入下载模式，运行时可正常使用
- UART1 默认 TX=GPIO0, RX=GPIO4，开箱即用，无需额外配置
