# 多模态视觉通用训练框架开发者文档（Fruit Vision Framework）

> **适用场景**：基于 RTX 5090 工作站完成多水果（樱桃、苹果、草莓等）品质分级模型的通用训练框架，支持 ConvNeXt/EfficientNet 等主流视觉骨干网络的训练、验证与 ONNX 导出

> **交付目标**：生成可部署至 2核2G 云端 C++ 后端的 ONNX 模型文件，支持多水果品类动态切换

> **前置依赖**：符合框架规范的分类数据集、RTX 5090 硬件环境、PyTorch 训练框架

> **核心特性**：
> - YAML 配置驱动，零代码切换水果品类
> - 全局去硬编码，类别数量与名称完全可配置
> - 统一文件树架构，支持多实验并行管理

# 3. YAML 配置驱动体系

## 3.1 配置概述

框架采用 **YAML 配置文件** 作为唯一数据源，替代所有硬编码参数。训练时只需指定配置文件路径，即可切换不同的水果品类和训练策略。

## 3.2 配置字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `task_name` | string | 是 | 任务/水果名称，用于生成输出目录和模型文件名 |
| `data_root` | string | 是 | 数据集根目录（框架自动拼接 train/val/test 子路径） |
| `num_classes` | int | 是 | 分类类别数量 |
| `class_names` | list[str] | 是 | 类别名称列表，顺序即索引顺序 |
| `train.batch_size` | int | 是 | 训练批次大小 |
| `train.epochs` | int | 是 | 训练轮数 |
| `train.lr` | float | 是 | 初始学习率 |
| `model.name` | string | 否 | 模型名称，默认 `convnext_tiny` |
| `model.pretrained` | bool | 否 | 是否使用预训练权重，默认 `true` |

## 3.5 配置加载代码

```python
# src/utils.py
import yaml
from pathlib import Path
from dataclasses import dataclass
from typing import List

@dataclass
class TrainConfig:
    batch_size: int
    epochs: int
    lr: float
    warmup_epochs: int = 5
    patience: int = 5
    weight_decay: float = 1e-5

@dataclass
class ModelConfig:
    name: str = "convnext_tiny"
    pretrained: bool = True

@dataclass
class ExperimentConfig:
    task_name: str
    data_root: str
    num_classes: int
    class_names: List[str]
    train: TrainConfig
    model: ModelConfig = None

    @classmethod
    def from_yaml(cls, config_path: str) -> "ExperimentConfig":
        with open(config_path, "r", encoding="utf-8") as f:
            config_dict = yaml.safe_load(f)

        train_config = TrainConfig(**config_dict.get("train", {}))
        model_config = ModelConfig(**config_dict.get("model", {})) if "model" in config_dict else ModelConfig()

        return cls(
            task_name=config_dict["task_name"],
            data_root=config_dict["data_root"],
            num_classes=config_dict["num_classes"],
            class_names=config_dict["class_names"],
            train=train_config,
            model=model_config
        )
```

# 4. 数据集准备

## 4.1 数据集目录规范

框架对数据集目录结构有统一要求，数据集根目录下必须包含 `train/`、`val/`、`test/` 三个子目录，每个子目录内按类别名称（由 YAML 配置的 `class_names` 指定）建立文件夹：

```
data/
└── {task_name}/                 # 例如 cherry, apple, strawberry
    ├── train/
    │   ├── grade_1/             # 类别名称必须与 YAML 配置完全一致
    │   ├── grade_2/
    │   └── ...
    ├── val/
    │   ├── grade_1/
    │   ├── grade_2/
    │   └── ...
    └── test/
        ├── grade_1/
        ├── grade_2/
        └── ...
```

> **重要**：类别文件夹名称必须与 YAML 配置中 `class_names` 的每一项完全匹配，框架会根据配置的顺序为每个类别分配索引（从 0 开始），而非按字母顺序排序。

## 4.2 数据预处理（严格对齐云端 C++ 逻辑）

预处理流程必须与云端 C++ 后端完全一致，否则会导致训练-推理精度偏差。

```python
# src/transforms.py
import torchvision.transforms as transforms

def get_train_transform():
    """训练集预处理（含数据增强）"""
    return transforms.Compose([
        # 1. 中心裁剪：640×480 → 480×480（与 C++ stb_image 裁剪逻辑一致）
        transforms.CenterCrop(480),
        # 2. 缩放至 ConvNeXt 标准输入：480×480 → 224×224
        transforms.Resize(224),
        # 3. 训练专属数据增强（提升泛化能力）
        transforms.RandomHorizontalFlip(p=0.5),
        transforms.RandomRotation(15),
        transforms.ColorJitter(brightness=0.1, contrast=0.1, saturation=0.1),
        # 4. 标准化（ImageNet 均值/方差，与 C++ 端对齐）
        transforms.ToTensor(),
        transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
    ])

def get_val_test_transform():
    """验证/测试集预处理（无增强，仅对齐 C++ 逻辑）"""
    return transforms.Compose([
        transforms.CenterCrop(480),
        transforms.Resize(224),
        transforms.ToTensor(),
        transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
    ])
```

> **工程红线**：CenterCrop 必须在 Resize 之前执行，此为保障训练-推理一致性的核心原则，不可颠倒。

## 4.3 数据集加载（FruitImageFolder）

框架提供了通用的 `FruitImageFolder` 数据集类，它从 YAML 配置中读取类别列表，动态构建类别映射，完全避免字典排序陷阱。

```python
# src/dataset.py
import os
from torchvision.datasets import ImageFolder
from typing import Tuple, List, Dict

class FruitImageFolder(ImageFolder):
    """
    通用水果图像数据集类。

    关键设计：从 YAML 配置读取 class_names，强制按配置顺序构建 class_to_idx，
    避免 ImageFolder 默认字典排序导致的类别映射错乱。
    """

    def __init__(self, root: str, transform, class_names: List[str]):
        """
        Args:
            root: 数据集子目录路径（如 train/, val/, test/）
            transform: 预处理 transform
            class_names: 从 YAML 配置读取的类别名称列表
        """
        self.root = root
        self.class_names = class_names
        super().__init__(root=root, transform=transform)

    def find_classes(self, directory: str) -> Tuple[List[str], Dict[str, int]]:
        """
        重写方法：从配置读取 class_names，强制按配置顺序分配索引。

        Args:
            directory: 目录路径（本实现中不使用，仅保持接口兼容）

        Returns:
            classes: 类别名称列表
            class_to_idx: 类别名称到索引的映射（按配置顺序）
        """
        # 直接使用初始化时传入的 class_names（来自 YAML 配置）
        classes = self.class_names
        class_to_idx = {cls_name: idx for idx, cls_name in enumerate(classes)}
        return classes, class_to_idx
```

# 5. 模型构建

## 5.1 模型选型依据

框架默认选用 ConvNeXt-Tiny 作为基线模型，理由如下：

1. 纯视觉 CNN 架构，细粒度特征提取能力优于 MobileNetV2 等轻量模型；
2. 预训练权重基于 ImageNet，适配农产品表面瑕疵检测场景；
3. 参数量适中（~28M），导出 ONNX 后体积约 110MB，适配云端 2核2G 内存限制。

> **可扩展性**：框架设计为支持多种骨干网络（EfficientNet、ResNet 等），只需在 `src/model.py` 中扩展对应的模型构建逻辑。

## 5.2 模型构建代码

```python
# src/model.py
import torch
import torch.nn as nn
from torchvision.models import convnext_tiny, efficientnet_b0, resnet50

def build_model(num_classes: int, model_name: str = "convnext_tiny", pretrained: bool = True):
    """
    动态构建分类模型，替换最后一层全连接层为指定类别数。

    Args:
        num_classes: 分类类别数量（从 YAML 配置读取）
        model_name: 模型名称，默认 convnext_tiny
        pretrained: 是否使用 ImageNet 预训练权重

    Returns:
        model: 适配指定类别数的模型
    """
    if model_name == "convnext_tiny":
        weights = "IMAGENET1K_V1" if pretrained else None
        model = convnext_tiny(weights=weights)
        # ConvNeXt-Tiny 的分类头：Sequential -> Linear -> LayerNorm -> Linear
        in_features = model.classifier[2].in_features
        model.classifier[2] = nn.Linear(in_features, num_classes)

    elif model_name == "efficientnet_b0":
        weights = "IMAGENET1K_V1" if pretrained else None
        model = efficientnet_b0(weights=weights)
        in_features = model.classifier[1].in_features
        model.classifier[1] = nn.Linear(in_features, num_classes)

    elif model_name == "resnet50":
        weights = "IMAGENET1K_V1" if pretrained else None
        model = resnet50(weights=weights)
        in_features = model.fc.in_features
        model.fc = nn.Linear(in_features, num_classes)

    else:
        raise ValueError(f"Unsupported model: {model_name}")

    return model
```

# 6. 模型训练

## 6.1 训练超参数配置

框架从 YAML 配置文件中读取所有超参数，不再在代码中硬编码。

| 超参数 | 配置字段 | 说明 |
|--------|----------|------|
| 学习率 | `train.lr` | 使用 OneCycleLR 调度器，包含 Warmup 阶段 |
| 优化器 | AdamW | 权重衰减通过 `train.weight_decay` 配置 |
| 损失函数 | CrossEntropyLoss | 类别均衡，无需加权 |
| 训练轮数 | `train.epochs` | 配合早停策略，避免过拟合 |
| 早停耐心值 | `train.patience` | 验证集精度连续 N 轮不提升则停止 |
| Warmup 轮数 | `train.warmup_epochs` | 前 N 轮逐步提升学习率，防止梯度爆炸 |

## 6.2 训练核心代码

```python
# src/trainer.py
import torch
import torch.nn as nn
import torch.optim as optim
from torch.optim.lr_scheduler import OneCycleLR
from tqdm import tqdm
from pathlib import Path

class Trainer:
    """训练器类：封装训练循环、验证、早停等逻辑"""

    def __init__(self, model, train_loader, val_loader, config):
        self.model = model
        self.train_loader = train_loader
        self.val_loader = val_loader
        self.config = config

        # 优化器与调度器（从配置读取）
        self.optimizer = optim.AdamW(
            model.parameters(),
            lr=config.train.lr,
            weight_decay=config.train.weight_decay
        )

        # OneCycleLR 配置
        total_steps = config.train.epochs * len(train_loader)
        warmup_steps = config.train.warmup_epochs * len(train_loader)
        self.scheduler = OneCycleLR(
            self.optimizer,
            max_lr=config.train.lr,
            total_steps=total_steps,
            pct_start=warmup_steps / total_steps,
            anneal_strategy='cos',
            div_factor=10,
            final_div_factor=1000
        )

        self.criterion = nn.CrossEntropyLoss().cuda()
        self.best_val_acc = 0.0
        self.early_stop_count = 0

        # 实验输出目录
        self.run_dir = Path("runs") / config.task_name
        self.run_dir.mkdir(parents=True, exist_ok=True)

    def train_one_epoch(self):
        self.model.train()
        total_loss = 0.0
        total_correct = 0
        total_samples = 0

        for images, labels in tqdm(self.train_loader, desc="Training"):
            images, labels = images.cuda(), labels.cuda()

            outputs = self.model(images)
            loss = self.criterion(outputs, labels)

            self.optimizer.zero_grad()
            loss.backward()
            self.optimizer.step()
            self.scheduler.step()

            total_loss += loss.item() * images.size(0)
            _, preds = torch.max(outputs, 1)
            total_correct += torch.sum(preds == labels.data)
            total_samples += images.size(0)

        return total_loss / total_samples, total_correct.double() / total_samples

    def validate(self):
        self.model.eval()
        total_loss = 0.0
        total_correct = 0
        total_samples = 0

        with torch.no_grad():
            for images, labels in tqdm(self.val_loader, desc="Validating"):
                images, labels = images.cuda(), labels.cuda()
                outputs = self.model(images)
                loss = self.criterion(outputs, labels)

                total_loss += loss.item() * images.size(0)
                _, preds = torch.max(outputs, 1)
                total_correct += torch.sum(preds == labels.data)
                total_samples += images.size(0)

        return total_loss / total_samples, total_correct.double() / total_samples

    def fit(self):
        """执行完整训练流程"""
        for epoch in range(self.config.train.epochs):
            print(f"\nEpoch {epoch+1}/{self.config.train.epochs}")

            train_loss, train_acc = self.train_one_epoch()
            val_loss, val_acc = self.validate()

            print(f"Train Loss: {train_loss:.4f} | Train Acc: {train_acc:.4f}")
            print(f"Val Loss: {val_loss:.4f} | Val Acc: {val_acc:.4f}")
            print(f"Current LR: {self.scheduler.get_last_lr()[0]:.6f}")

            # 早停逻辑
            if val_acc > self.best_val_acc:
                self.best_val_acc = val_acc
                self.early_stop_count = 0
                model_path = self.run_dir / "best_model.pth"
                torch.save(self.model.state_dict(), model_path)
                print(f"保存最优模型，验证集精度：{self.best_val_acc:.4f}")
            else:
                self.early_stop_count += 1
                if self.early_stop_count >= self.config.train.patience:
                    print(f"早停触发，停止训练（最优验证精度：{self.best_val_acc:.4f}）")
                    break

        return self.best_val_acc
```

# 9. RTX 5090 训练调优指南

## 9.1 显存优化

- 启用 `torch.backends.cudnn.benchmark = True`，提升卷积运算效率；
- 批次大小（Batch Size）建议设为 256/512，充分利用 RTX 5090 32GB 显存；
- 训练时启用梯度累积（如 `accumulation_steps=2`），等效增大批次大小且不增加显存占用；
- 启用混合精度训练，进一步降低显存占用，提升训练速度。

## 9.2 混合精度训练

```python
# 在 Trainer.__init__ 中添加混合精度支持
from torch.cuda.amp import autocast, GradScaler

class Trainer:
    def __init__(self, model, train_loader, val_loader, config):
        self.model = model
        self.train_loader = train_loader
        self.val_loader = val_loader
        self.config = config
        self.scaler = GradScaler()  # 混合精度梯度缩放器

    def train_one_epoch(self):
        self.model.train()
        total_loss = 0.0
        total_correct = 0
        total_samples = 0

        for images, labels in self.train_loader:
            images, labels = images.cuda(), labels.cuda()

            # 使用 autocast 启用混合精度
            with autocast():
                outputs = self.model(images)
                loss = self.criterion(outputs, labels)

            self.optimizer.zero_grad()
            self.scaler.scale(loss).backward()
            self.scaler.step(self.optimizer)
            self.scaler.update()
            self.scheduler.step()

            total_loss += loss.item() * images.size(0)
            _, preds = torch.max(outputs, 1)
            total_correct += torch.sum(preds == labels.data)
            total_samples += images.size(0)

        return total_loss / total_samples, total_correct.double() / total_samples
```

## 9.3 性能调优

- 关闭不必要的进程（如浏览器、IDE），避免抢占 GPU 资源；
- 数据集放入 NVMe SSD，提升数据读取速度，避免 CPU 瓶颈导致 GPU 闲置；
- 启用 cudnn.benchmark：`torch.backends.cudnn.benchmark = True`

# 11. 常见问题排查

| 问题现象 | 排查方向 |
|----------|----------|
| 训练时 GPU 利用率低 | 1. 数据加载线程数（num_workers）不足；2. 批次大小过小；3. 未启用 cudnn 优化；4. 数据集读取速度慢（非 NVMe 存储） |
| 验证集精度远低于训练集 | 1. 过拟合（增加数据增强/权重衰减）；2. 预处理与 C++ 端不一致；3. 类别映射错误 |
| ONNX 导出后推理报错 | 1. 输入输出节点名不匹配；2. opset_version 过高/过低；3. 模型未设为 eval 模式；4. 训练与推理预处理不一致 |
| RTX 5090 显存溢出 | 1. 批次大小过大（适当降低至 256）；2. 未释放中间变量；3. 启用梯度累积/混合精度训练；4. 关闭其他占用显存的进程 |
| 初始轮次 Loss 跑飞、无法收敛 | 1. 未启用学习率 Warmup；2. 初始学习率过高；3. 数据预处理错误；4. 类别映射错乱 |
| GPU 无法被 PyTorch 识别 | 1. CUDA 版本与 RTX 5090 不兼容（需 ≥12.4）；2. PyTorch 未安装 CUDA 版本；3. 显卡驱动未更新 |
| 类别索引与预期不符 | 1. YAML 配置中 class_names 顺序与文件夹名称不匹配；2. 未使用 FruitImageFolder；3. 数据集文件夹名称拼写错误 |
