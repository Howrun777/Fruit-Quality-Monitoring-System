## 一、项目概述

### 1.1 项目简介

SmartCherry 是一个基于多光谱传感器的**大樱桃无损糖度检测平台**，网页端负责数据可视化展示与后台管理功能。项目采用前后端分离架构，前端通过 RESTful API 与后端数据处理服务器通信，实现实时监控与历史数据分析。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| **官网首页** | 产品介绍页，含动画粒子效果、视差滚动、数据统计动画 |
| **管理后台** | 数据可视化大屏，实时展示传感器数据 |
| **登录系统** | Cookie + Session 认证的安全登录流程 |
| **数据图表** | ECharts 驱动的多维数据可视化 |
| **自动轮询** | 产区/设备自动切换与数据定时刷新 |

### 1.3 技术栈

| 类别 | 技术选型 |
|-----|---------|
| **核心框架** | 原生 HTML5 + CSS3 + JavaScript (ES6+) |
| **图表库** | ECharts 5.x (`assets/libs/echarts.min.js`) |
| **字体** | Google Fonts (Cinzel, Playfair Display, DM Sans, DM Mono) |
| **通信方式** | Fetch API + RESTful JSON |
| **状态管理** | 原生 JS 模块化变量 (window 全局挂载) |
| **后端地址** | `http://47.107.41.102:9000` |

## 四、管理后台 (admin/)

### 4.1 登录模块 (login.html)

**登录流程：**
```
用户输入账号密码
    ↓
调用 login(username, password)
    ↓
POST /api/admin/login
    ↓
成功: 浏览器自动处理 Set-Cookie → 跳转 dashboard.html
失败: 显示后端返回的错误信息
```

**关键代码：**

```javascript
// admin/js/api.js
const BASE_URL = "http://47.107.41.102:9000/api";

async function login(username, password) {
    const response = await fetch(`${BASE_URL}/admin/login`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ username, password })
    });
    
    const resData = await response.json();
    
    if (resData.code === 200) {
        // Cookie 由浏览器自动处理
        window.location.href = "dashboard.html";
    } else {
        showError(resData.msg);
    }
}
```

**登录页 UI 特性：**
- 密码显隐切换
- 输入验证（空值检测）
- 错误状态高亮
- 回车键快捷提交
- 忘记密码提示弹窗

### 6.1 后端地址

### 6.2 接口列表

| 接口路径 | 方法 | 描述 | 返回格式 |
|---------|------|------|---------|
| `/api/admin/login` | POST | 管理员登录 | `{code, msg, ...}` |
| `/api/admin/field/list` | GET | 获取产区列表 | `{code, data: {list: [{field_id}]}}` |

| `/api/admin/fruit/history` | GET | 设备历史数据（**含AI视觉快照**） | `{code, data: [{timestamp, sugar_brix, quality_level, image_url}]}` |
| `/api/admin/field/environment` | GET | 产区环境数据 | `{code, data: [{timestamp, temperature, humidity, light, spoilage}]}` |

### 6.3 AI视觉快照数据说明

AI视觉快照数据通过 `/api/admin/fruit/history` 接口返回，历史数据中包含以下与视觉快照相关的字段：

| 字段名 | 类型 | 描述 |
|-------|------|------|
| `timestamp` | int | Unix时间戳（秒） |
| `sugar_brix` | float | 糖度值 (°Brix) |
| `quality_level` | int | 品质等级 (0-10) |
| `image_url` | string | 快照图片相对路径 |

**前端处理逻辑：**

```javascript
// 从 fruit/history 数据中筛选并转换视觉快照格式
const visionData = jsonFruit.data
    .filter(d => d.quality_level !== undefined && d.image_url)
    .map(d => {
        const date = new Date(d.timestamp * 1000);
        const monthDay = `${(date.getMonth()+1).toString().padStart(2,'0')}.${date.getDate().toString().padStart(2,'0')}`;
        const timeStr = `${date.getHours().toString().padStart(2,'0')}:${date.getMinutes().toString().padStart(2,'0')}`;
        return {
            imgUrl: `http://47.107.41.102:9000/${d.image_url}`,  // 拼接完整URL
            quality: d.quality_level >= 0 ? d.quality_level : 'N/A',
            time: `${monthDay}-${timeStr}`
        };
    });

renderVisionGallery(visionData);
```

**数据格式转换：**

```
后端原始数据                    →  前端展示格式
─────────────────────────────────────────────────
image_url: "assets/..."       →  imgUrl: "http://47.107.41.102:9000/assets/..."
quality_level: 8              →  quality: 8
timestamp: 1712800000         →  time: "4.29-23:15"
```

## 十、常见问题

### Q1: 图表不显示？
- 检查 ECharts 库是否正确加载
- 确认容器有明确高度
- 调用 `chart.resize()` 重新计算尺寸

### Q2: 数据不刷新？
- 检查浏览器控制台网络请求
- 确认后端服务正常运行
- 检查 `reloadAllChartsData` 函数无 JS 异常

### Q3: 跨域问题？
- 前后端部署在不同域名时需配置 CORS
- 或通过 Nginx 反向代理合并到同域

### Q4: 登录失效？
- 检查 Cookie 是否被浏览器拦截
- 确认 Session 有效期设置
