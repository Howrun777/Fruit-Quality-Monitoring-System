【系统总体需求】
本系统由 FPGA 作为主控，外挂 OV5640 摄像头、数码管显示器，并通过 SPI 总线连接 ESP32 模块实现图像的无线传输。
【工作流程及详细规格】
1. 指令接收与解析：
FPGA 通过 UART（或明确具体接口） 接收外部设备的触发指令。
蜂鸣器功能: 如果接收到时间戳就会响一声持续2秒
数据格式为 ASCII 字符串（以 \n 结尾）：CAM|<Device_ID>|<Token>|<Timestamp>\n（例：CAM|1001-01-01|token_001|1712800000\n）。
FPGA 需解析提取出这三个字段。
2. 时间显示：
FPGA 将提取的 UNIX Timestamp 转换为 北京时间（UTC+8） 的 HH.MM.SS（时.分.秒）格式。
将结果实时显示在 FPGA 外接的数码管上。
3. 图像采集：
解析成功后，FPGA 通过 SCCB/I2C 配置并触发 OV5640 摄像头。
OV5640 输出一张 640x480 分辨率的 JPEG 格式图片，FPGA 需对其进行缓存。
4. FPGA -> ESP32 数据传输：
FPGA 通过 SPI 接口 将数据发往 ESP32。
传输的数据包需包含：Device_ID、Token、Timestamp 以及 JPEG 图像文件数据（因为ESP32发网络请求需要这些鉴权信息）。
5. ESP32 -> 服务器上传 (HTTP)：
ESP32 通过 Wi-Fi 连接网络。
接口地址：POST http://47.107.41.102:9000/api/device/vision
Header 要求：
Device_ID: <设备ID>
Token: <鉴权Token>
Timestamp: <时间戳>
Content-Type: application/octet-stream
Body 要求：纯 JPEG 文件的二进制流数据。
