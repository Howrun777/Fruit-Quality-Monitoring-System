# 无人机模块开发者文档

## 一、概述

本文档定义了无人机模块与后端服务器之间的通信接口规范。

### 业务背景

服务器端在计算腐败情况时，如果腐败程度超过60%，就把樱桃信息加入快腐败队列。无人机模块会不断向服务器请求当前腐败队列是否存在快腐败樱桃，如果发现有快腐败樱桃，就获取樱桃信息并执行语言广播。

---

## 二、基础约定

| 项目 | 说明 |
|------|------|
| 通信协议 | HTTP/1.1 |
| 数据格式 | 请求/响应均为 JSON |
| 服务端地址 | `http://47.107.41.102:9000` |
| 设备编号规则 | 产区号-组号-设备号（例：`1001-01-01`） |
| 状态码 | 200=成功，401=认证失败，400=数据校验失败，500=服务器内部错误 |

---

## 三、认证方式

所有接口请求需要在HTTP Header中携带设备认证信息：

```
device_id: 1001-01-01
token: device-token-001
```

---

## 四、接口设计

### 4.1 查询快腐败樱桃队列

**接口用途**：检查队列中是否存在快腐败樱桃，如果有则返回樱桃信息。

- **请求方法**：`GET`
- **请求路径**：`/api/drone/fast-decay/queue`
- **是否需要认证**：是（device_id + token）

**请求示例**：

```
GET /api/drone/fast-decay/queue
device_id: 1001-01-01
token: device-token-001
```

**成功响应（200 OK）**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "hasFastDecayCherries": true,
    "cherries": [
      {
        "deviceId": "1001-01-01",
        "timestamp": "2026-05-02T14:20:00Z",
        "decayRate": 48.5
      },
      {
        "deviceId": "1001-01-02",
        "timestamp": "2026-05-02T13:45:00Z",
        "decayRate": 47.2
      }
    ]
  }
}
```

**无快腐败樱桃时的响应**：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "hasFastDecayCherries": false,
    "cherries": []
  }
}
```

**响应字段说明**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| hasFastDecayCherries | boolean | 是 | 队列中是否存在快腐败樱桃 |
| cherries | array | 是 | 快腐败樱桃列表（无数据时为空数组） |
| cherries[].deviceId | string | 是 | 监测设备ID（无人机可根据此ID解析出设备位置） |
| cherries[].timestamp | string | 是 | 监测到快腐败状态的时间戳（ISO 8601格式） |
| cherries[].decayRate | float | 是 | 当前腐败程度百分比（如48.5表示48.5%） |

**失败响应**：

- 401：`{"code":401,"msg":"认证失败：Token错误、设备不存在或已被禁用"}`

---

## 五、数据格式规范

### 5.1 时间格式

所有时间字段统一使用ISO 8601格式，示例：

```
2026-05-02T14:20:00Z
```

---

## 六、接口调用建议

### 6.1 轮询频率

建议无人机每30秒查询一次快腐败队列。

### 6.2 重试机制

建议实现简单的重试机制，失败后等待2秒重试，最多重试3次。

---

## 七、广播内容参考

无人机获取到快腐败樱桃信息后，可使用以下内容进行语言广播：

```
"注意！检测到樱桃腐败，请相关人员及时处理。设备编号：{deviceId}，腐败程度：{decayRate}%。"
```

---

## 八、关键业务规则

| 规则 | 说明 |
|------|------|
| 腐败预警阈值 | spoilage > 60% → 加入快腐败队列 |
| 快腐败樱桃标识 | 腐败程度接近阈值但尚未超标的樱桃 |
| 队列过期机制 | 建议队列数据保留一定时间后自动清除 |
