# 智能图像采集与无线传输系统：详细功能模块开发者文档

## 系统需求

### 1. 系统总体需求

本系统由 FPGA 作为主控，外挂 OV5640 摄像头、数码管显示器，并通过 SPI 总线连接 ESP32 模块实现图像的无线传输。

### 2. 工作流程及详细规格

#### 2.1 指令接收与解析

FPGA 通过 UART 接收外部设备的触发指令。

蜂鸣器功能：如果接收到时间戳就会响一声持续2秒。

数据格式为 ASCII 字符串（以 `\n` 结尾）：

```
CAM|<Device_ID>|<Token>|<Timestamp>
```

例：`CAM|1001-01-01|token_001|1712800000`

FPGA 需解析提取出这三个字段。

#### 2.2 时间显示

FPGA 将提取的 UNIX Timestamp 转换为北京时间（UTC+8）的 HH.MM.SS（时.分.秒）格式，并将结果实时显示在 FPGA 外接的数码管上。

#### 2.3 图像采集

解析成功后，FPGA 通过 SCCB/I2C 配置并触发 OV5640 摄像头。OV5640 输出一张 640x480 分辨率的 JPEG 格式图片，FPGA 需对其进行缓存。

#### 2.4 FPGA -> ESP32 数据传输

FPGA 通过 SPI 接口将数据发往 ESP32。传输的数据包需包含：Device_ID、Token、Timestamp 以及 JPEG 图像文件数据（因为ESP32发网络请求需要这些鉴权信息）。

#### 2.5 ESP32 -> 服务器上传 (HTTP)

ESP32 通过 Wi-Fi 连接网络。

接口地址：`POST http://47.107.41.102:9000/api/device/vision`

Header 要求：

- `Device_ID: <设备ID>`
- `Token: <鉴权Token>`
- `Timestamp: <时间戳>`
- `Content-Type: application/octet-stream`

Body 要求：纯 JPEG 文件的二进制流数据。

## 模块架构总览

本系统代码结构推荐采用**自顶向下 (Top-Down)** 的模块化设计，顶层模块（Top Module）负责实例化以下 6 个子模块，并完成它们之间的数据流互联。

1. `uart_cmd_parser` (指令接收与解析模块)
2. `buzzer_pwm_ctrl` (蜂鸣器提示音控制模块)
3. `rtc_display_74hc595` (时间换算与数码管串行驱动模块)
4. `ov5640_jpeg_capture` (摄像头配置与 JPEG 帧捕获模块)
5. `sdram_frame_buffer` (图像跨时钟域 SDRAM 缓存模块)
6. `spi_master_tx` (自定义协议打包与 SPI 发送模块)

*（注：ESP32 侧固件作为独立工程开发）*

---

## 模块 1：指令接收与解析模块 (`uart_cmd_parser`)

- **功能描述**：通过串口接收 PC 或外部主控发来的触发字符串，解析出鉴权信息与时间戳，并生成全局触发信号。
- **输入/输出信号**：
  - `Input`: `sys_clk` (50MHz), `sys_rst_n`, `uart_rx` (PIN_N6)
  - `Output`: `cmd_valid` (指令有效脉冲), `device_id`, `token`, `timestamp` (时间戳寄存器输出)
- **实现逻辑 (FSM 状态机)**：
  1. **IDLE**：等待 UART 接收完成标志位。当匹配到字符 `C`、`A`、`M`、`|` 序列时，进入数据提取状态。
  2. **EXTRACT_ID**：将接收到的字符存入 `device_id` 移位寄存器，遇到 `|` 切换状态。
  3. **EXTRACT_TOKEN**：将字符存入 `token` 移位寄存器，遇到 `|` 切换状态。
  4. **EXTRACT_TIME**：将时间戳 ASCII 码转为 HEX 数字存入 `timestamp`（如将 "1712800000" 转为 32-bit 整数）。
  5. **DONE**：遇到 `\n` (0x0A)，校验数据完整性，拉高 `cmd_valid` 一个时钟周期，随后返回 IDLE。

## 模块 2：蜂鸣器提示音控制模块 (`buzzer_pwm_ctrl`)

- **功能描述**：接收到指令有效信号后，驱动开发板上的**无源蜂鸣器**发出 2 秒的清脆提示音。
- **输入/输出信号**：
  - `Input`: `sys_clk` (50MHz), `sys_rst_n`, `cmd_valid` (触发信号)
  - `Output`: `beep` (PIN_J11, PWM 输出)
- **实现逻辑**：
  - **2 秒定时器**：检测到 `cmd_valid` 脉冲时，启动一个计数器，目标值为 `100,000,000`（50MHz 下为 2 秒）。
  - **PWM 生成**：在计数器运行的这 2 秒内，由另一个子计数器生成频率约 **3kHz**（计数周期约 16666）、占空比 **50%** 的方波，赋值给 `beep` 引脚。
  - 2 秒到达后，计数器清零，`beep` 强制拉低（静音）。

## 模块 3：时间换算与数码管串行驱动模块 (`rtc_display_74hc595`)

- **功能描述**：将 32-bit 的 UNIX 时间戳转换为北京时间的 HH.MM.SS，并通过 74HC595 移位寄存器点亮 6 位数码管。
- **实现逻辑**：
  1. **时区与算术模块**：
    - `Timestamp_BJ = Timestamp + 28800`。
    - `Current_Seconds = Timestamp_BJ % 86400`（当天走过的秒数）。
    - 使用分步除法/状态机，算出 `Hours` ( / 3600), `Minutes` ( / 60 % 60), `Seconds` ( % 60)。并拆分为 6 个十进制个位数（如 12 时拆为 1 和 2）。
  2. **74HC595 串行移位扫描 (1kHz 定时)**：
    - 准备一个 16-bit 寄存器：`[8位段选码 (数字字模)] + [8位位选码 (选择哪一位数码管)]`。
    - **Shift 状态**：拉低 `shcp`，将最高位推入 `ds`，拉高 `shcp`，循环 16 次。
    - **Latch 状态**：给 `stcp` 一个高脉冲，将 16 位数据锁存输出。
    - **使能**：维持 `oe` 为低电平。每隔 1ms 切换下一位数码管的数据，利用人眼视觉暂留实现 `HH.MM.SS` 显示。

## 模块 4：摄像头配置与 JPEG 帧捕获模块 (`ov5640_jpeg_capture`)

- **功能描述**：通过 I2C/SCCB 初始化 OV5640 摄像头输出 JPEG 流，并精准提取单帧图像的有效载荷。
- **实现逻辑**：
  1. **SCCB Master**：系统复位后，向 OV5640 的 I2C 地址 (写 0x78, 读 0x79) 依次发送初始化寄存器数组（配置为 640x480, JPEG 压缩格式）。
  2. **JPEG 边界检测**：
    - 由于 JPEG 每帧大小不一，无法依靠行列计数判断结束。
    - 在 `href` 为高的 `pclk` 节拍下，实时监控 `data[7:0]` 流。
    - **帧起始**：检测到连续两字节为 `0xFF` 紧接着 `0xD8`，标记帧开始。
    - **帧结束**：检测到连续两字节为 `0xFF` 紧接着 `0xD9`，标记一帧结束。
  3. 输出 `jpeg_data_en` 和 `jpeg_data` 字节流，并统计出当前帧的**总字节数** `file_size`。

## 模块 5：图像跨时钟域 SDRAM 缓存模块 (`sdram_frame_buffer`)

- **功能描述**：解决摄像头输出速度（PCLK）与 SPI 传输速度（SCK）不匹配的问题。
- **实现逻辑**：
  - **写端口 (Camera 端)**：将模块 4 输出的 JPEG 字节流打包成 16-bit，存入一个写 FIFO。FIFO 水线到达阈值时，向 SDRAM 发起 Burst Write 请求，存入 SDRAM 指定 Bank。
  - **读端口 (SPI 端)**：当检测到模块 6 请求发送数据时，从 SDRAM 发起 Burst Read 请求，存入读 FIFO，供 SPI 发送端平滑读取。

## 模块 6：自定义协议打包与 SPI 发送模块 (`spi_master_tx`)

- **功能描述**：将鉴权数据和 SDRAM 中的图片数据组装成数据包，通过 SPI 发送给 ESP32。
- **实现逻辑**：
  1. 检测到完整的一帧 JPEG 已写入 SDRAM 且接收到触发指令后，拉低 `spi_cs_n`。
  2. 按照严格顺序将数据 Shift Out 至 `spi_mosi` 引脚：
    - 发送 `0xAA 0xBB`（2 字节包头）
    - 发送 `Device_ID` 长度及数据
    - 发送 `Token` 长度及数据
    - 发送 `Timestamp` 长度及数据
    - 发送 `file_size` （4 字节大端格式）
    - 切换至 **图像流发送状态**，持续从 SDRAM 读 FIFO 中取数据并由 SPI 发送，直到发送字节数等于 `file_size`。
  3. 拉高 `spi_cs_n` 结束传输。

## 模块 7：ESP32-C3 固件逻辑设计 (C/C++ 开发)

### ESP32-C3 SPI 引脚定义

> **重要说明**：原始文档未规定 ESP32-C3 侧 SPI 引脚，以下为推荐配置。

#### 推荐方案（HSPI 接口）


| 功能                  | ESP32-C3 GPIO | 说明                   |
| ------------------- | ------------- | -------------------- |
| **SPI 时钟 (SCK)**    | GPIO4         | HSPI 专用时钟，可直接连接 FPGA |
| **SPI MOSI (主收从发)** | GPIO5         | 从 FPGA 接收数据          |
| **SPI MISO (主发从收)** | GPIO6         | 固定为输入                |
| **SPI 片选 (CS)**     | GPIO7         | 接受 FPGA 控制           |


#### 引脚选择理由


| 对比项     | VSPI (GPIO0/1/9/10) | HSPI (GPIO2/3/6/10)   |
| ------- | ------------------- | --------------------- |
| GPIO0/1 | 通常用于烧录/启动模式，容易冲突    | GPIO2/3 是普通 GPIO，灵活性高 |
| 推荐度     | ⚠️ 谨慎使用             | ✅ **推荐使用**            |


#### FPGA 侧对应引脚（保持不变）


| 功能                   | FPGA 引脚 | 说明   |
| -------------------- | ------- | ---- |
| SPI 时钟               | PIN_A10 | 保持不变 |
| SPI MOSI (发往 ESP32)  | PIN_B10 | 保持不变 |
| SPI MISO (从 ESP32 收) | PIN_A9  | 保持不变 |
| SPI 片选               | PIN_B9  | 保持不变 |


#### 完整 SPI 连接对照表


| 功能              | FPGA 侧  | ESP32-C3 侧         |
| --------------- | ------- | ------------------ |
| SPI 时钟          | PIN_A10 | GPIO4 (SCK)        |
| SPI 主发从收 (MOSI) | PIN_B10 | GPIO5 (MOSI)       |
| SPI 主收从发 (MISO) | PIN_A9  | GPIO6 (MISO)       |
| SPI 片选          | PIN_B9  | GPIO7 (CS)         |
| 共地              | GND     | GND                |

### ESP32-C3 固件核心工作流

- **功能描述**：作为 SPI Slave 接收 FPGA 传来的拼装流，剥离数据并发送 HTTP 网络请求。
- **核心工作流**：
  1. **SPI DMA 接收**：由于可能涉及几十 KB 的图像数据，必须使用 ESP32 的 **SPI Slave DMA** 模式。
  2. **协议状态机解析**：
    - 在内存 Buffer 中搜索 `0xAA 0xBB`。
    - 按约定偏移量依次读取 `ID`、`Token`、`Timestamp` 字符串。
    - 读取 `file_size` 变量。
    - 将剩余的 `file_size` 长度的内容锁定为 **纯 JPEG Buffer**。
  3. **HTTP POST 构建**：
    ```c
      esp_http_client_config_t config = {
          .url = "http://47.107.41.102:9000/api/device/vision",
          .method = HTTP_METHOD_POST,
      };
      esp_http_client_handle_t client = esp_http_client_init(&config);

      // 动态注入 Header (利用之前提取的变量)
      esp_http_client_set_header(client, "Device_ID", device_id_str);
      esp_http_client_set_header(client, "Token", token_str);
      esp_http_client_set_header(client, "Timestamp", timestamp_str);
      esp_http_client_set_header(client, "Content-Type", "application/octet-stream");

      // 挂载 JPEG 二进制流
      esp_http_client_set_post_field(client, jpeg_buffer, file_size);

      // 发送并清理
      esp_err_t err = esp_http_client_perform(client);
    ```
